int nomp_init(int argc, const char **argv)

Initializes libnomp with the specified backend, platform, device, etc.

Initializes nomp code generation for the specified backend (e.g., OpenCL, Cuda, etc) using command line arguments. Also platform id, device id, verbose level, annotation scripte and annotation function can be specified as well. Returns a negative value if an error occurs during the initialization, otherwise returns 0. Calling this method twice (without nomp_finalize in between) will return an error (but not segfault). Currently only supports Cuda and OpenCL backends.

Accepted arguments:

  • -b|--backend <backend-name> Specify backend type (Default: opencl).

  • -p|--platform <platform-index> Specify platform id (Default: 0).

  • -d|--device <device-index> Specify device id (Default: 0).

  • -v|--verbose <verbose-level> Specify verbose level (Default: 0).

  • -as|--annts-script <annotation-script> Specify the directory which the annotation script resides.

  • -af|--annts-func <annotation-function> Specify the annotation funtion name.

Example usage: 
const char *argv[] = {"--backend", "opencl", "-device", "0", "--platform",
"0"};
int argc = 6;
int err = nomp_init(argc, argv);

Parameters

  • argc – [in] The number of arguments to nomp_init().

  • argv – [in] Arguments as strings, values followed by options.

Returns

int

int nomp_update(void *ptr, size_t start_idx, size_t end_idx, size_t unit_size, int op)

Performs device to host (D2H) and host to device (H2D) memory transfers, allocating and freeing of memory in the device.

Operation op will be performed on the array slice [start_idx, end_idx), i.e., on array elements start_idx, … end_idx - 1. This method returns a non-zero value if there is an error and 0 otherwise.

Example usage: 

int N = 10;
double a[10];
for (unsigned i = 0; i < N; i++)
  a[i] = i;

// Copy the value of `a` into device
int err = nomp_update(a, 0, N, sizeof(double), NOMP_TO);

// Execution of a kernel which uses `a`
...

// Copy the updated value of `a` from device
int err = nomp_update(a, 0, N, sizeof(double), NOMP_FROM);

// Free the device memory allocated for `a`
int err = nomp_update(a, 0, N, sizeof(double), NOMP_FREE);

Parameters

  • ptr – [in] Pointer to the host memory location.

  • start_idx – [in] Start index in the vector to start copying.

  • end_idx – [in] End index in the vector to end the copying.

  • unit_size – [in] Size of a single vector element.

  • op – [in] Operation to perform (One of Update Direction).

Returns

int

int nomp_jit(int *id, const char *c_src, const char **clauses)

Generate and compile a kernel for the targe backend (OpenCL, etc.) from C source.

Target backend is the one provided during the initialization of libnomp using nomp_init(). User defined code transformations will be applied based on the clauses specified in clauses argument. Additional kernel meta data can be passed using the clauses as well.

Example usage: 

int N = 10;
double a[10], b[10];
for (unsigned i = 0; i < N; i++) {
  a[i] = i;
  b[i] = 10 -i
}
const char *knl = "for (unsigned i = 0; i < N; i++) a[i] += b[i];"
static int id = -1;
const char *clauses[4] = {"transform", "file", "function", 0};
int err = nomp_jit(&id, knl, clauses);

Parameters

  • id – [out] Id of the generated kernel.

  • c_src – [in] Kernel source in C.

  • clauses – [in] Clauses to provide meta information about the kernel.

Returns

int

int nomp_run(int id, int nargs, ...)

Runs the kernel generated by nomp_jit().

Runs the kernel with a given kernel id. Kernel id is followed by the number of arguments (i.e., variables). Then for each argument, four values has to be passed. First, is the argument name as a string. Second is the argument type (one of Data Types). Third is the sizeof argument and the fourth is the pointer to the actual argument itself.

Parameters

  • id – [in] Id of the kernel to be run.

  • nargs – [in] Number of arguments to the kernel.

  • ... – [in] Four values mentioned above for each argument.

Returns

int

int nomp_finalize(void)

Finalizes libnomp runtime.

Frees allocated runtime resources for libnomp. Returns a non-zero value if an error occurs during the finalize process, otherwise returns 0. Calling this method before nomp_init() will retrun an error. Calling this method twice will also return an error.

Returns

int

Map Direction

Defines the operation direction in nomp_update.

NOMP_ALLOC

Allocate memory on the device.

NOMP_TO

Copy host data to device. Memory will be allocated if not allocated already.

NOMP_FROM

Copy device data to host.

NOMP_FREE

Free memory allocated on the device.
